# Тестовые документы для развертывания

Эта директория содержит набор из ~1000 тестовых документов для демонстрации работы системы и развертывания на сервере.

## Содержимое

В директории находятся разнообразные типы юридических документов Республики Казахстан (~100 документов, ~7 MB):

- **Бюджетные документы** (~15) - решения о бюджетах различных уровней
- **Постановления Правительства** (~15) - нормативные акты правительства РК
- **Указы Президента** (~10) - указы главы государства
- **Решения маслихатов** (~20) - решения местных представительных органов
- **Комментарии** (~10) - комментарии к нормативным актам
- **Законы** (~5) - законы Республики Казахстан
- **Приказы** (~5) - приказы министерств и ведомств
- **Распоряжения** (~5) - распоряжения различных органов
- **Положения** (~5) - положения о различных процедурах
- **Инструкции** (~5) - инструкции и методические указания
- И другие типы документов

## Использование

Эти документы используются для:
1. **Демонстрации** работы системы RAG с большим объемом данных
2. **Тестирования** функционала поиска и генерации ответов
3. **Развертывания** на сервере без необходимости загружать полный корпус документов (1364+ документов)
4. **Разработки и отладки** - достаточно документов для полноценного тестирования

## Добавление документов

Для добавления большего количества документов используйте скрипт:

```bash
# Добавить ~1000 документов из базы данных
python3 scripts/add_test_documents.py

# Или подготовить небольшой набор примеров
python3 scripts/prepare_examples.py
```

Скрипт автоматически:
- Выбирает разнообразные типы документов
- Исключает уже скопированные файлы
- Распределяет документы по категориям

## Важно

- Основная директория `current/` исключена из git через `.gitignore`
- В git попадают только файлы из `current/examples/`
- Для продакшена рекомендуется загрузить полный корпус документов через админ-панель

## Обработка документов

После развертывания на сервере:

1. Загрузите документы через админ-панель (`/admin`)
2. Или используйте автоматическую настройку системы
3. Документы будут обработаны и добавлены в базу данных с embeddings
