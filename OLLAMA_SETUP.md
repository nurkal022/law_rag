# Настройка Ollama с моделью gpt-oss:20b

## Быстрая настройка

### Вариант 1: Автоматический скрипт (рекомендуется)

```bash
./scripts/setup_ollama_gptoss.sh
```

Скрипт автоматически:
- Проверит, запущена ли Ollama
- Проверит наличие модели gpt-oss:20b
- Протестирует модель
- Настроит переменные окружения

### Вариант 2: Ручная настройка

#### 1. Убедитесь, что Ollama запущена

```bash
# Проверка статуса
curl http://localhost:11434/api/tags

# Если не запущена, запустите:
ollama serve
```

#### 2. Проверьте наличие модели

```bash
ollama list
```

Если модели нет, скачайте её:
```bash
ollama pull gpt-oss:20b
```

#### 3. Настройте переменные окружения

Создайте или отредактируйте файл `.env` в корне проекта:

```bash
LLM_PROVIDER_TYPE=ollama
LLM_MODEL=gpt-oss:20b
OLLAMA_BASE_URL=http://localhost:11434
```

#### 4. Перезапустите приложение

```bash
python app.py
```

### Вариант 3: Через веб-интерфейс

1. Откройте админ-панель: http://localhost:5003/admin
2. Перейдите в раздел "Настройки моделей LLM"
3. Выберите "Ollama (Локальный)"
4. Укажите модель: `gpt-oss:20b`
5. URL сервера: `http://localhost:11434`
6. Нажмите "Сохранить настройки"

## Проверка работы

### Тест через curl

```bash
curl http://localhost:11434/api/generate -d '{
  "model": "gpt-oss:20b",
  "prompt": "Что такое закон?",
  "stream": false
}'
```

### Тест через приложение

1. Откройте чат: http://localhost:5003/chat
2. Задайте вопрос
3. Проверьте, что ответ генерируется локальной моделью

## Оптимизация для больших моделей

Модель gpt-oss:20b требует около 13-14 GB памяти. Для лучшей производительности:

### 1. Используйте GPU (если доступно)

Ollama автоматически использует GPU, если он доступен. Проверьте:

```bash
ollama show gpt-oss:20b
```

### 2. Настройте параметры генерации

В файле `config.py` можно настроить:

```python
MAX_TOKENS = 2000  # Уменьшите для быстрых ответов
TEMPERATURE = 0.1  # Низкая температура для точных ответов
```

### 3. Увеличьте таймаут (если нужно)

В `llm_providers/ollama_provider.py` таймаут установлен на 300 секунд (5 минут), что должно быть достаточно.

## Устранение проблем

### Ollama не запускается

```bash
# Проверьте, не запущена ли уже
ps aux | grep ollama

# Запустите в фоне
nohup ollama serve > ollama.log 2>&1 &

# Или в отдельном терминале
ollama serve
```

### Модель не отвечает

1. Проверьте доступность Ollama:
   ```bash
   curl http://localhost:11434/api/tags
   ```

2. Проверьте логи Ollama:
   ```bash
   tail -f ollama.log
   ```

3. Перезапустите Ollama:
   ```bash
   pkill ollama
   ollama serve
   ```

### Медленные ответы

- Модель 20B параметров требует значительных ресурсов
- Используйте GPU для ускорения
- Уменьшите `MAX_TOKENS` в конфигурации
- Рассмотрите использование более легкой модели для быстрых ответов

## Альтернативные модели

Если gpt-oss:20b слишком медленная, попробуйте:

```bash
# Более легкие модели
ollama pull llama3.2        # ~2GB
ollama pull qwen2.5:7b      # ~4.5GB
ollama pull mistral:7b      # ~4GB

# Затем измените LLM_MODEL в .env
```
