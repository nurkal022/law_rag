## Обновление документов

Для обновления набора документов:

```bash
# Добавить еще ~100 документов
python3 scripts/add_test_documents.py

# Или подготовить небольшой набор примеров
python3 scripts/prepare_examples.py
```

## Для развертывания на сервере

После клонирования репозитория:

1. **Разверните документы:**
   ```bash
   # Автоматически скопирует документы из examples в current/
   python3 scripts/deploy_documents.py
   
   # Или вручную:
   mkdir -p current
   cp current/examples/*.txt current/
   ```

2. **Обработайте документы:**
   - Перейдите в админ-панель: `/admin`
   - Нажмите "Обработать все документы"
   - Дождитесь завершения обработки (создание embeddings)

3. **Проверьте статус:**
   - В админ-панели должна быть статистика: документы, чанки, embeddings
   - После обработки чат будет работать

## Размер репозитория

- Код: ~несколько MB
- Тестовые документы: ~4-5 MB (ограничено до 50 MB)
- **Итого:** ~10-15 MB

Это оптимальный размер для git репозитория с тестовыми данными - достаточно для демонстрации, быстро загружается и обрабатывается.

## Ограничение размера

Для удобного коммита в git размер документов ограничен до **50 МБ**. Если нужно уменьшить размер:

```bash
# Автоматически ограничит до 50 МБ
python3 scripts/limit_size.py

# Или используйте reduce_examples.py (тоже ограничивает размер)
python3 scripts/reduce_examples.py
```
